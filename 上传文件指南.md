# 上传网站文件到服务器指南

## 方法一：使用FTP工具（推荐，简单易用）

### 推荐工具：FileZilla（免费）

#### 1. 下载安装FileZilla
- 访问：https://filezilla-project.org/
- 下载并安装 FileZilla Client

#### 2. 连接服务器

打开FileZilla后：
1. 点击左上角"文件" → "站点管理器"
2. 点击"新站点"
3. 填写信息：
   - **协议**：选择 `SFTP`（如果支持）或 `FTP`
   - **主机**：填写您的服务器IP地址
   - **端口**：SFTP用22，FTP用21
   - **用户名**：服务器用户名（通常是 `root`）
   - **密码**：服务器密码
   - **登录类型**：选择"正常"
4. 点击"连接"

#### 3. 上传文件

连接成功后：
1. **左侧窗口**：本地电脑，找到项目中的 `dist` 文件夹
2. **右侧窗口**：服务器，导航到网站目录（通常是 `/var/www/html` 或 `/usr/share/nginx/html`）
3. **上传步骤**：
   - 在左侧选中 `dist` 文件夹内的所有文件
   - 拖拽到右侧窗口
   - 等待上传完成

## 方法二：使用WinSCP（Windows用户推荐）

### 1. 下载安装WinSCP
- 访问：https://winscp.net/
- 下载并安装

### 2. 连接服务器
- 填写服务器IP、用户名、密码
- 协议选择：SFTP
- 点击"登录"

### 3. 上传文件
- 左侧：本地 `dist` 文件夹
- 右侧：服务器网站目录
- 拖拽上传

---

## 方法三：使用SCP命令（命令行）

### Windows PowerShell 或 CMD

```powershell
# 上传整个dist文件夹内容到服务器
scp -r dist/* root@您的服务器IP:/var/www/html/
```

### Linux/Mac 终端

```bash
# 上传整个dist文件夹内容到服务器
scp -r dist/* root@您的服务器IP:/var/www/html/
```

**注意**：需要替换：
- `root` 为您的服务器用户名
- `您的服务器IP` 为实际IP地址
- `/var/www/html/` 为实际的网站目录路径

## 上传后的文件权限设置（Linux服务器）

上传完成后，SSH登录服务器执行：

```bash
# 设置文件权限
sudo chmod -R 755 /var/www/html

# 设置文件所有者（如果需要）
sudo chown -R www-data:www-data /var/www/html
# 或
sudo chown -R nginx:nginx /var/www/html
```

## 常见问题

### Q1: 不知道服务器网站目录在哪里？
**A:** 常见路径：
- Nginx: `/usr/share/nginx/html` 或 `/var/www/html`
- Apache: `/var/www/html` 或 `/var/www/`

### Q2: 上传后无法访问？
**A:** 检查：
1. 文件是否上传到正确目录
2. 文件权限是否正确
3. Web服务器是否运行

### Q3: 连接服务器失败？
**A:** 检查：
1. 服务器IP是否正确
2. 用户名密码是否正确
3. 防火墙是否允许SSH/FTP连接
4. 服务器是否开启SSH/FTP服务
