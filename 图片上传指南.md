# 图片上传指南

## 📁 图片存放位置

网站中的图片有两种存放方式，我已经为您创建好了目录结构：

### 方式一：public 目录（推荐用于案例图片）

**路径：** `public/images/`

**子目录：**
- `public/images/cases/` - 案例展示图片
- `public/images/logo/` - Logo和品牌图片  
- `public/images/laozi/` - 老子爷爷IP相关图片
- `public/images/partners/` - 合作伙伴Logo

**使用方法：**
1. 直接将图片文件复制到对应目录
2. 在代码中使用绝对路径：`/images/cases/图片名.jpg`

**优点：**
- 不需要导入，直接使用路径即可
- 适合大量图片
- 图片路径简单明了

### 方式二：src/assets 目录（推荐用于Logo等少量图片）

**路径：** `src/assets/images/`

**使用方法：**
1. 将图片放入 `src/assets/images/` 目录
2. 在代码中导入：
```tsx
import logo from '@/assets/images/logo.png'
```

**优点：**
- 会被Vite打包优化
- 可以自动处理图片优化

## 📝 如何替换现有占位图片

### 步骤1：上传图片文件

将您的图片文件放入对应目录，例如：
- 案例图片 → `public/images/cases/case-1.jpg`
- Logo → `public/images/logo/logo.png`
- 老子爷爷形象 → `public/images/laozi/laozi-character.png`

### 步骤2：修改代码中的图片路径

#### 案例1：首页案例展示（Home.tsx）

找到 `src/pages/Home.tsx` 文件，修改 cases 数组：

```tsx
const cases = [
  {
    title: '我与宪法-郑州市统计局政策',
    category: '二维Flash+MG动画',
    image: '/images/cases/case-1.jpg'  // 替换这里
  },
  {
    title: 'VR360全景视频拍摄-印尼美娜多的日出横屏',
    category: 'VR360全景视频',
    image: '/images/cases/case-2.jpg'  // 替换这里
  },
  // ... 其他案例
]
```

#### 案例2：案例展示页面（Cases.tsx）

找到 `src/pages/Cases.tsx` 文件，修改 allCases 数组中的 image 字段。

#### 案例3：老子爷爷形象（Original.tsx）

找到 `src/pages/Original.tsx` 文件，可以这样修改：

```tsx
// 方式一：使用 public 目录
<div className="laozi-image">
  <img src="/images/laozi/laozi-character.png" alt="老子爷爷" />
</div>

// 方式二：使用 assets 目录
import laoziImage from '@/assets/images/laozi-character.png'
// ...
<div className="laozi-image">
  <img src={laoziImage} alt="老子爷爷" />
</div>
```

## 🖼️ 图片命名建议

**Logo：**

**老子爷爷：**

## 📐 图片尺寸建议

| 用途 | 推荐尺寸 | 格式 |
|------|---------|------|
| 案例缩略图（首页） | 400x300px 或 800x600px | JPG |
| 案例详情图 | 1200x800px 或更大 | JPG |
| Logo | 根据需求，建议200x200px以上 | PNG（透明背景） |
| 老子爷爷形象 | 500x500px 或更大 | PNG |
| 合作伙伴Logo | 200x100px | PNG |

## 🎨 图片格式说明

- **JPG/JPEG**: 适合照片、复杂图像（文件小，加载快）
- **PNG**: 适合Logo、图标、需要透明背景的图片
- **WebP**: 现代格式，文件更小（推荐，但需要转换）
- **SVG**: 适合图标、Logo（矢量图，可无限缩放）

## ⚡ 图片优化建议

1. **压缩图片**：上传前使用工具压缩图片（如 TinyPNG）
2. **使用合适格式**：照片用JPG，Logo用PNG
3. **控制文件大小**：单张图片建议不超过500KB
4. **提供多种尺寸**：为不同场景准备不同尺寸的图片

## 📌 注意事项

1. **路径以 `/` 开头**：使用 public 目录时，路径必须以 `/` 开头
2. **文件名避免中文**：虽然支持，但建议使用英文文件名避免编码问题
3. **大小写敏感**：Linux服务器上路径大小写敏感，建议统一使用小写
4. **刷新缓存**：如果图片不显示，尝试硬刷新（Ctrl+F5）

## 🆘 常见问题

**Q: 图片上传后不显示？**
A: 检查路径是否正确，确保图片文件在正确的目录中，路径以 `/` 开头

**Q: 图片太大，加载慢？**
A: 使用图片压缩工具（如 TinyPNG）压缩后再上传

**Q: 需要透明背景的Logo？**
A: 使用PNG格式，确保图片本身有透明背景

**Q: 如何批量替换图片？**
A: 可以一次性将所有图片放入对应目录，然后批量修改代码中的路径
